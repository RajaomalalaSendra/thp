Cours : CSS

1. Introduction

Les MotherFuckingWebsites c'est bien, mais comment organiser une page web complexe avec un header, un footer, des éléments centrés, des sections ? Cette ressource va te permettre d'aller plus loin dans l'univers du CSS, l'outil indispensable du développeur qui veut mettre en page son site web, rajouter des couleurs, personnaliser la police et bien d'autres choses encore!

2. Historique

CSS est sorti au milieu des années 90, avec une technique révolutionnaire de mise en page par cascade. C'est devenu rapidement le langage universel de mise en page d'une page web.

3. La ressource

3.1. Apprendre FlexBox

Pour cette ressource c'est assez simple : nous allons te demander de continuer le cours HTML / CSS de OpenClassrooms et de reprendre là où tu t'étais arrêté. Avance donc dans la track HTML / CSS de OpenClassrooms, jusqu'à la partie "Mise en page avec FlexBox" inclue.

Maintenant que tu as compris comment marche FlexBox, nous allons te proposer un jeu qui te permettra de le pratiquer: fais donc FlexBox Froggy. C'est un excellent exercice pour t'entraîner rapidement sur la mise en forme avec FlexBox.

3.2. Récap sur FlexBox

FlexBox est maintenant LA référence en CSS pour centrer des éléments, les répartir en hauteur ou en largeur, etc. Pour ceux qui avaient déjà pratiqué le CSS auparavant, évitez au maximum les inline-block ou autre float. FlexBox doit être privilégié car compatible avec tous les navigateurs récents et son rendu est plus uniforme quel que soit la taille des écrans (de l'écran plat jusqu'au smartphone).

Pour mettre en place FlexBox, rien de plus simple! Créez un élément HTML(par exemple une div) contenant les autres éléments à aligner en FlexBox. Exemple: 

<div id="conteneur">
	<div>Elément 1 à afficher </div>
	<div>Elément 2 à afficher </div>
	<div>Elément 3 à afficher </div>
</div>

Puis rajoute-lui les propriétés CSS suivantes:

  1. Applique le FlexBox à tous les éléments enfants avec le CSS display: flex;
  2. Choisis l'axe d'alignement des éléments enfants: sur une ligne (par défaut) avec flex-direction: row; ou en colonne flex-direction: column;
  3. Choisis ensuite comment les éléments doivent se répartir sur cet axe:
    • Tous collés au début avec justify-content: flex-start;?
    • Tous collés à la fin avec justify-content: flex-end;?
    • Tous centrés avec justify-content: center;?
    • Répartis sur l'axe sans marge avec le bord avec justify-content: space-between;?
    • Répartis sur l'axe + une marge avec le bord avec justify-content: space-around;?
  4. Enfin choisis si tu souhaites également faire un alignement selon l'autre axe (par exemple tu as réparti les éléments horizontalement mais souhaite aussi les aligner verticalement) avec la commande align-items: *qqch*;

⚠️ ALERTE ERREUR COMMUNE

Si tu veux répartir horizontalement tes éléments avec FlexBox, pas de souci : le navigateur va utiliser toute la largeur disponible sur l'écran. Par contre pour répartir verticalement, tu dois absolument spécifier une hauteur à l'élément conteneur, par exemple avec height: 100px;. Sans ça, tes justify-content: center ou justify-content: space-around n'auront aucun effet verticalement !

Je te vois venir avec tes MAIS POURQUOI ? Et bien tout simplement car verticalement, on peut scroller sur un site web. Donc quand on dit au navigateur "centre-moi ces éléments verticalement", s'il ne sait pas sur quelle hauteur il travaille, comment en trouver le centre? 😘

🚀 ALERTE BONNE ASTUCE

Bosser en FlexBox c'est pas simple: on change le CSS, on rafraîchit le navigateur, les éléments bougent mais on sait pas trop comment ni pourquoi...

Pour faciliter la lisibilité de tout ça, mets temporairement des bordures à tous tes éléments en leur appliquant une class= "border-flex" à laquelle vous appliquez le css suivant (au format "border: *epaisseur_de_bordure* *style_de_bordure* *couleur_de_bordure*") :

.border-flex {
	border: 2px solid red;
}

La présence de ces bordures va légèrement décaler tes éléments (elles font quelques pixels) mais c'est super pour voir quelle taille fait chaque élément, où il se place, qui est centré sur quoi, etc.

4. Points importants à retenir

La mise en page d'une page web, c'est aussi simple qu'une leçon OpenClassrooms couplée à un petit jeu de grenouilles en délire.

5. Pour aller plus loin

Pas besoin de passer des années dessus, mais tu peux finir le cours de OpenClassrooms sur HTML / CSS si tu veux être le champion du front !

Tu peux aussi faire la suite de FlexBox Froggy, Grid Garden qui te présente la notion de FlexGrid.

Cours : Pair-Programming

1. Introduction

Le pair programming est une technique qui permet de bien s'améliorer en code, utilisée par de nombreuses startups. Le concept est simple : deux personnes sont sur un seul ordinateur, l'une d'entre elles tient le clavier et tape, l'autre lui donne son avis. Au final les deux codent et prennent les décisions ensemble. Ensuite, elles alternent régulièrement (à chaque exercice ou commit par exemple).

Nous allons voir quelques guidelines pour faire du pair programming à The Hacking Project.

2. Historique

Le pair programming est une technique assez récente pour faire du code, utilisée principalement en entreprise agile (le code sera plus propre) et en école (génial pour apprendre).

3. La ressource

3.1. Prendre 2 personne de niveau proche

S'il y a une disparité trop grande entre les niveaux (surtout quand on débute la programmation), une personne survolera tandis que l'autre soit ne suivra pas, soit se verra offrir la solution sur un plateau sans galérer (et du coup sans apprendre). Il faut donc que les deux soient, si possible, à niveau équivalent.

  • Les groupes de bons pourront défricher le terrain pour mieux aider les autres sur les problématiques qu'ils vont rencontrer. "-Hey tu sais comment faire ceci ?" "-Ah oui, on a eu le même souci on a pas mal galéré mais en fait la solution c'est ça"
  • Les moins à l'aise vont parler le même langage et essayer de résoudre eux même les problématiques sur lesquelles ils bloquent (c'est en bloquant qu'on apprend !!)

3.2. Créer un repo Github commun

Le repo GitHub va te permettre de basculer le code d'un ordi à l'autre. Pour cela, il va falloir que tu fasses des git clone, git push, git pull. N'hésite pas à revenir au cours de Git qui explique cela plus en détails.

3.3. Utiliser un seul ordinateur pour 2

Une personne va taper le code sur son ordinateur en fonction de ce que les deux décident de faire. L'échange est au cœur de cet exercice ! Il faut discuter de votre compréhension des problèmes, débattre des meilleures façons de le résoudre, être attentif aux fautes de l'autre et au final produire du code à deux.

3.4. Alterner

Il est important d'alterner d'ordinateur et donc de personne qui tape. Celui qui écrivait peut alors git push son code, et son partenaire fait un git pull pour récupérer les modifications. Vous alternez et changez d'ordinateur.

4. Points importants à retenir

Pour faire du pair programming efficace, il faut deux personnes de niveau similaire qui travaillent sur un seul ordinateur, en alternant entre celui qui écrit et celui qui dicte.

Cette technique est au cœur de la méthodologie THP d'apprentissage en groupe : essayez de vous y tenir autant que possible. L'ensemble des projets validants doivent être faits en pair-programming.

⚠️ ALERTE ERREUR COMMUNE

Les personnes ayant un bon niveau de code ont tendance à ne pas voir l'intérêt du pair-programming : on file le clavier à quelqu'un de moins à l'aise et alors qu'on sait exactement quoi taper, lui met trois plombes à trouver où sont les accolades "{ }" … 😤

Pourtant c'est une erreur ! Coder dans son coin c'est l'occasion d'aider son camarade à comprendre mais surtout de s'aider soi-même en se forçant à expliquer ce qu'on fait et pourquoi on le fait. C'est une méthode très efficace pour comprendre en profondeur un concept et surtout le retenir ! Ça porte même un nom : la méthode Feynman.

5. Pour aller plus loin

L'article Wikipédia explique pas mal les avantages et défauts du pair-programming, ainsi que les différents types de pair-programming.

Projet : Faire la page d'accueil de Google

1. Introduction

Suite aux succès des projets de conseil de ton groupe, un entrepreneur plein d'avenir est venu vers vous pour vous pitcher son concept d'entreprise : une page très simple avec un formulaire où les utilisateurs pourront faire une recherche, qui trouvera sur les internets les sites mentionnant cette recherche. Il y a quelque chose de révolutionnaire dans cette idée donc vous vous chauffez à en réaliser la page d'accueil.

Comme votre client n'a pas vraiment l'air de connaître les entreprises un peu #hype du moment, vous vous êtes dits qu'il suffirait de reprendre la page d'accueil du moment et lui montrer. Il n'y verra que du feu, et à vous la gloire 😎

Le projet d'aujourd'hui est donc simple en apparence : reproduire la page d'accueil de Google.

2. Le projet

2.1. Mise en place

Avant toute chose, nous te conseillons de préparer le repository Github, afin que ton groupe puisse partager son code entre chacun ton projet dedans et nous le soumettre une fois le projet fini.

Pour ce projet, il est OBLIGATOIRE de le faire en pair-programming.

2.2. Au boulot

Ta mission est donc de refaire la page d'accueil de Google (celle avec juste la barre de recherche, sans être connecté). Voici quelques conseils :

  • Commence par mettre juste les éléments principaux (form, boutons, logo), puis positionne-les convenablement (FlexBox, padding et margin), avant d'appliquer le CSS pour leur apparence (bordure, couleur, hover, etc.).
  • Sois méthodique en reproduisant petit à petit la page de haut en bas. Commence par faire les icônes du haut, puis le logo central, puis la barre de recherche, puis les 2 boutons "recherche" et "j'ai de la chance", etc.
  • Google est ton ami. Rien de tel qu'une bonne vieille recherche how to center a div pour savoir comment centrer une div.
  • Dans le doute, fais une div.
  • Ne sois pas perfectionniste. Déjà tu n'as pas vu Javascript donc pas besoin de se prendre la tête avec du JS, et pas besoin de refaire la page au quart de pixel près. Le but est de te faire comprendre la mise en page d'une page web en CSS.
  • Pour la navbar du haut (ce tuto explique très bien comment une navbar fonctionne.
  • Dans le doute, rajoute une div.
  • Le footer devrait être similaire à la navbar du haut.
  • Et voilà ! Tu n'as plus qu'à pousser ton projet sur Github, remplir le formulaire et siroter un thé.

2.3. Mets en ligne ton œuvre sur GitHub Pages

Pour que tout internet (et l'entrepreneur qui te paye!) puisse profiter de ton œuvre, mets en ligne ta page via GitHub Pages! Pour ce faire, rien de plus simple:

  1. Push sur le repo ton fichier html sous le nom "index.html" ;
  2. N'oublie pas de mettre dans le repo le fichier .css contenant ton taff de mise en page ;
  3. Configure ton repo pour qu'il mette automatique en ligne une version GitHub Pages à chaque push sur la branche "master". En théorie, je devrais te laisser galérer un peu à chercher comment faire sur Google (pour que le métier rentre !💪) mais cette fois-ci je vais te filer un coup de main.

3. Rendu attendu

Un repo GitHub qui contient le code de ton clone de Google.

Ton repo doit contenir à la racine un fichier "README.md" contenant:

  • Le nom des 2 personnes ayant bossé sur le projet en pair-programming
  • Des blagues et emoji pour amadouer tes correcteurs et les inciter à booster ta note 🤡
  • Un lien vers la version en production sur GitHub Pages
